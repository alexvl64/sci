# CLAUDE.md

## Project Overview
SparkCore Investment OÜ (Estonian AIFM) marketing website. Static HTML site with PHP backend for forms/proxy and Python scripts that generate blog HTML from Markdown sources.

## Tech Stack & Architecture
- **Frontend**: Vanilla JS, Tailwind CSS 3.4 (utility-only — no custom classes)
- **i18n**: Client-side EN/FR via `data-i18n` / `data-i18n-html` / `data-i18n-placeholder` attributes + `assets/js/translations.json`
- **Blog pipeline**: `MD/*.md` (Markdown + YAML frontmatter) → `scripts/build_*.py` → `blog/*.html`
- **Backend**: PHP 7+ (`form_send.php`, `proxy.php`, `secure_pdf.php`); SQLite3 rate-limiting in `form_send.php`
- **Charts**: ApexCharts pulling live data from Google Sheets JSON API
- **Animations**: AOS.js (`data-aos` attributes)

## Essential Commands
```bash
# Compile Tailwind CSS (only build step)
npm run build:css
# → reads assets/css/input.css, config at assets/js/tailwind.config.js
# → outputs minified assets/css/tailwind.min.css

# Regenerate a blog article from Markdown
python3 scripts/build_mica_casp_article.py
python3 scripts/build_white_label_aifm_article.py
```
No test runner, no linter, no dev server configured.

## Code Conventions
- **JS**: camelCase variables/functions; kebab-case HTML IDs (`close-sidebar`, `btn-en-nav`)
- **HTML IDs**: pattern `{element}-{location}` e.g. `btn-fr-footer`
- **Tailwind colors**: camelCase custom names — `darkGray`, `steelBlue`, `softBlueGray`, `lightBlue`, `charcoal`, `cream` (defined in `assets/js/tailwind.config.js`)
- **Python**: snake_case variables, UPPER_CASE module-level constants (`INTERNAL`, `FAQ`, `ROOT`)
- **PHP**: camelCase variables

## What NOT To Do
- **Do not hand-edit `blog/*.html`** — they are generated by `scripts/build_*.py`; edit the source in `MD/` then regenerate
- **`MD/` is in `.gitignore`** — do not commit Markdown source files
- **Do not add CSS classes** outside Tailwind utilities; all styling is utility-first with no custom class layer
- **Do not modify `assets/css/tailwind.min.css`** directly — it is a compiled output; edit `input.css` and rebuild
- **Tailwind config lives in `assets/js/`**, not the project root — pass `--config ./assets/js/tailwind.config.js` on every CLI invocation (already in the npm script)
- **No build system for HTML** — root/`fr/`/`factsheets/` HTML files are hand-maintained; only `blog/` is generated
